# 测试问题集合

这份题集用于覆盖当前项目的主要能力、典型链路和高风险问法，方便部署后验证：

- `planner` 是否稳定输出合理步骤
- `metadata` 参数是否被实际执行节点使用
- `route / traffic / service / rag / report` 是否命中正确分支
- 复合问题最终是否走综合回答，而不是只套单一模板
- 相对时间、OD 识别、路网报表、服务区配套、政策解释等场景是否回答自然

建议每题至少观察 4 个点：

1. `Planner final execution plan`
2. 实际调用的工具参数
3. `Answer prompt selected`
4. 最终回复是否聚焦用户真实问题

## 使用建议

- 单轮问题先逐条跑一遍，确认规划和参数稳定。
- 复合问题建议重点看 `step_results` 是否都产出。
- 报表类问题建议同时检查是否输出表格。
- 时间相关问题建议确认回答里是否能体现当前时间上下文。
- 对比类问题建议准备一轮“上一条 assistant 回答”再继续提问。

## 十一、多轮对比测试

这组题需要连续提问，重点看上下文利用是否自然。

1. 先问：`请输出今天浙江全省高速路况汇总表`
2. 再问：`基于上次结果，再生成一版对比表格`
3. 再问：`重点说一下变化最大的几条高速`

观察点：

- 第二轮是否命中 `report`
- 是否利用到上一轮回答
- 第三轮是否能围绕上一轮上下文继续总结

## 十二、推荐部署回归顺序

如果时间有限，建议按下面顺序跑：

1. `R03` 苍南去玉环怎么走
2. `RT02` 苍南去玉环堵不堵
3. `T04` 吕塘角枢纽入口情况如何
4. `T03` 雷甸口子大客车限行了吗
5. `S05` 杭州到金华沿途有没有充电桩
6. `K03` 今天上高速到明天下高速要过路费吗
7. `N02` 请给我今天全路网路况对比表格
8. `C01` 杭州到金华堵不堵，沿途有没有服务区充电桩

这 8 题能比较高效地覆盖：

- OD 抽取
- metadata 优先级
- route -> traffic
- route -> service
- rag 政策解释
- report 表格输出
- 复合问题综合回答
